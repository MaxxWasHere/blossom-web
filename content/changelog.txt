# Blossom 2.3.7

## Fixed
- Sidebar selection — visiting the Macro Schedule tab no longer strips the active color off your previous sidebar category; the active highlight reappears correctly when you navigate back. Schedule now takes over the sidebar via CSS (like the Music page) instead of removing React's active class, which React wouldn't restore on a same-category click.
- Macro Schedule (W.I.P.) — selecting a timeline block now deselects it when you click anywhere outside the editor (page header, WIP banner, page background) instead of leaving it accent-colored; the deselect listener was previously scoped only to the editor.
- Macro Schedule (W.I.P.) — fixed a document keydown/click listener leak that accumulated each time the Schedule tab was reopened.
- Settings & extras → Logs & diagnostics: **Open logs folder** now reliably opens the logs folder in File Explorer — it previously opened behind the app window (or did nothing) because the folder was launched from a background API thread that Windows blocks from claiming the foreground.
- Appearance → custom UI: the **Open themes folder** button gets the same fix and opens the themes folder reliably.

## UI
- Macro Schedule (W.I.P.) editor polish — taller, comfier timeline with a softer ruler; per-block node dots are muted by default and light up accent on hover/selection/live for clearer hierarchy; selected blocks lift with a stronger accent ring.
- Macro Schedule interaction — profile cards, palette chips, zoom and properties controls now show keyboard focus rings and M3 pressed-state springs; timeline blocks are keyboard-focusable (Tab) with Enter to edit and Delete to remove; the empty-state dropzone and "Drag to add" palette label get affordance icons. Still clearly labeled Work in Progress.

# Blossom 2.3.5

## New
- Music player — open it from the new **Music** sidebar category, which shows the player as a full in-app page (everything fits inside: now-playing, transport, seek/volume/speed, a scrolling playlist and the add-track section). Add tracks by pasting a web URL or browsing a local audio file (mp3, wav, ogg, m4a, flac…), then play/pause, seek, skip, shuffle and repeat. Adjust volume, mute, and playback speed (0.5×–2×), reorder or remove tracks. The page opens/closes with spring scale/fade motion, the sidebar selection animates like the other categories, and the player closes when you pick another category (audio keeps playing in the background). Closed by default on launch; your playlist, volume, speed, loop, shuffle and last track save to your config and restore when you reopen the player (no auto-play on launch; local files keep their path in config, not a data URL).
- Animated background — Appearance has a new **Animated background** card with live wallpaper presets (Aurora, Mesh, Stars, Bubbles), plus a **Custom media** preset that uses your own video, gif or image — paste a web URL or **Browse** a local file (local media is streamed back through the app's server, so large videos work without base64). Opacity, blur, dim and speed sliders tune the look; speed controls video playback rate. The background sits behind cards, the toolbar, inputs and sidebar items, so most surfaces stay solid and text stays readable; it freezes to a static frame (and videos pause) when Reduce motion is on. Off by default.

## UI
- Expressive motion everywhere — inputs, selects, checkboxes, chips, badges, secondary buttons, titlebar controls, list/table rows, links and code now share the same Material 3 Expressive easing: hover lifts, press springs, focus glows, and a checked pop. Dialogs, toasts, the command palette and the license gate get container-transform entrances; injected Blossom panels (Appearance, Animated background, Fishing, Merchant, Biome webhooks) fade-rise on mount. All gated on Appearance → Reduce motion.

# Blossom 2.1.0

## New
- Added Fishing Mode. Detects bite, plays reel minigame, walks to dock, optional sell. Enable it in the Macro tab and calibrate Fishing and Movements first.
- Added a Macro Calibrations hub. One page for movement, merchant, potions, and fishing calibrations instead of separate cards.

## Improved
- Reworded in-app copy and docs: intro, credits, changelog, and README.
- Credits and docs note the lineage. Blossom is a fork of Coteab (by Vapure), which came from the original Noteab biome macro.
- Reorganized project files. Python code lives under `src/`, docs under `docs/`. Run the app with `py run_local_ui.py` from the repo root.
- Trimmed README and docs for download-and-run setup. Release links, calibration, and webhooks are up front.
- Smaller download and lighter install. The vision component used by fishing is no longer packed into the exe. It downloads once on first use of Fishing Mode and is cached in your local app data, which cuts the installer size by about 47 MB.
- Fishing uses built-in walk paths to the dock and seller. Drop `fishing.json` or `fishing_sell.json` in your paths folder for custom routes. `fishing_playback_multiplier` sets replay speed.
- The fishing vision component is verified by checksum before it is used, and any tampered or partial cached copy is re-downloaded. If it cannot be fetched, fishing falls back to NumPy and still works, just slightly slower.
- Auto Merchant uses the original detection flow again: E key presses, dialogue clicks, Mari/Jester/Rin OCR, then buys from the item list.
- Moved the top toolbar closer to the title bar.
- Cards now highlight on hover across the whole card, not just the header strip.
- Lighter idle cost. The macro now reuses its loaded settings and only re-reads the config file from disk when you actually change a setting, instead of re-parsing it several times a second while running.
- Biome and aura detection now reuses the current Roblox log file and only rescans the logs folder every few seconds instead of every second, lowering background CPU and disk use while the app sits open.
- Built-in update banner hiding now runs through the shared page watcher instead of its own always-on DOM observer, so the UI does less per-change work.

## Fixed
- Aura Detection webhooks work again on the Auras tab: rarity threshold, force-ping list, and Discord user ID match the original Noteab behavior, using Blossom embed styling.
- Aurora and Eggland removed from the Biome Notifier list.
- The Start / Stop header button no longer flickers. Hotkey labels and icons update without fighting the icon replacer.
- The potion name field on Potion Crafting no longer bounces or loses focus while you type. The add panel stays put instead of being re-inserted every frame.

## Macro
- Potion Switching stays off until Auto Potion Craft is on. The UI and macro both ignore the switcher when craft is off.
- With Auto Potion Craft on, webhook currency screenshots are disabled and the Currency Screenshot calibration section is hidden.
- While Fishing Mode runs, merchant, quests, potion craft, obby, BR/SC, and biome selector pause. If Auto Potion Craft is also on, crafting takes priority and fishing waits until craft is off.
- The first time Fishing Mode runs it may download the vision component before starting. This happens once, then it is cached. Without it, fishing uses the NumPy fallback.
- Auto Merchant detection is faster. Name OCR stops on the first Mari/Jester/Rin match and skips the sleep after the last retry.
- Auto Merchant calibrations now cover all five shop slots, purchase amount, Set Max, dialogue, open/close, and purchase, each with Mark buttons in the hub. Buy flow clicks the amount field, then Max if calibrated, then purchase.
- Auto Merchant Set Max is a single shared calibration again. The hub has one Set max Mark button used by every non-Rin merchant, and the buy flow clicks it for Mari and Jester. Rin has no Max button and skips Max entirely. Any older per-merchant Mari/Jester Max calibrations still work as a fallback.
- The macro runs one UI action per tick on an interval timer with a fixed priority order (buffs, merchant, SC, BR, biome selector, potion, quests, obby), so enabled actions take turns instead of colliding or starving each other. Idle ticks skip work because the config is only re-read when it changes on disk.
- Starting the macro resets all per-run state and brings up every enabled action from a clean slate. Merchant, quests, potion craft, and obby each run once at start, fishing and the biome watcher come up, and the rest begin on their own timers.
- Added an in-app installer for the fishing vision component on the Settings and extras page. It shows whether the component is installed, installs it on demand with a progress bar and percent, and verifies the download by checksum before use. If the component is not published yet or the download fails, it says so and fishing keeps working with the built-in fallback. Restart the app after updating so the new installer loads.
- The app now checks for the optional vision component on boot and installs it in the background if it is missing, so fishing is ready without a manual install. The checksum verification and built-in fallback are unchanged, and a failed or offline check never blocks launch.
- Added Auto Eden detection, ported from the original Noteab macro. When enabled, it watches the in-game chat for the "Eden has appeared" announcement using OCR (with the same spoof guard against players faking the line and a 40 minute cooldown), then pings your Discord webhook with a chat screenshot. It uses your existing chat OCR calibrations.

# Blossom 2.0.0

## Overview
